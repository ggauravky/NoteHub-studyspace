🧠 Computational Thinking and Programming

1. Introduction to Problem Solving

Problem-solving involves understanding the problem, identifying the inputs and outputs, and designing a step-by-step solution.

Computational Thinking

It involves four key techniques:

1. Decomposition: Breaking a large problem into smaller, manageable parts.

2. Pattern Recognition: Finding similarities or trends within problems.

3. Abstraction: Focusing on essential information while ignoring irrelevant details.

4. Algorithm Design: Developing step-by-step instructions to solve the problem.

2. Algorithms and Flowcharts

Algorithm: A finite sequence of precise, step-by-step instructions to solve a problem.

Flowchart: A graphical representation of an algorithm using standard symbols.

Flowchart Symbols

Oval: Start / Stop

Parallelogram: Input / Output

Rectangle: Process / Calculation

Diamond: Decision (Yes/No)

Arrows: Flow of control

3. Pseudocode

An informal, high-level description of an algorithm. It uses structural conventions of a programming language but is intended for human reading, not machine execution.

4. Introduction to Python

Python is a high-level, interpreted, interactive, and object-oriented scripting language. Created by Guido van Rossum.

Modes of Python

Interactive Mode: Code is typed at the Python prompt (>>>) and executed immediately. Good for testing small snippets.

Script Mode: Code is written in a file (with .py extension) and executed as a whole program.
